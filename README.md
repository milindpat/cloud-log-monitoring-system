# Cloud Log Monitoring & Alert System
A Python-based real-time authentication log monitoring tool that detects repeated failed login attempts and generates structured alert artifacts for security analysis workflows.
This project simulates how Security Operations Center (SOC) monitoring systems detect brute-force login activity and produce actionable threat intelligence outputs.

- Real-time authentication log monitoring
- Brute-force attack detection using configurable thresholds
- Automated alert generation and CSV reporting
- Structured JSON threat intelligence export

## How It Works

The script continuously monitors a server log file for login activity.  
When multiple failed login attempts are detected from the same IP address, the system triggers an alert and records the attack.

- A security alert is generated
- The attack is recorded in CSV format
- The suspicious IP is stored
- A structured JSON threat intelligence file is generated
- A summary report of the attack session is created

## Output Files

### output/alerts.txt

Stores timestamped alerts generated by the monitoring system.

Example:
```
[2026-03-28 16:24:46] ALERT: 5 failed login attempts detected from IP 192.168.1.10
```


### output/attack_report.csv

Logs detected attacks in CSV format.

Example:
```
192.168.1.77,5
192.168.1.22,5
```


### output/suspicious_ips.txt

Stores unique IP addresses that triggered alerts.

Example:
```
192.168.1.77
192.168.1.22
```


### output/suspicious_ips.json

Structured JSON export of suspicious IP intelligence suitable for SIEM pipelines or automation workflows.

Example:
```
{
  "suspicious_ips": ["192.168.1.10"],
  "total_suspicious_ips": 1,
  "generated_at": "2026-03-28 16:24:46"
}
```


### output/attack_summary.txt

Summary report containing total suspicious IPs detected, total failed login attempts observed,
most aggressive attacker IP, and highest number of failed attempts from a single source.

Example:
```
Attack Summary
--------------
Total suspicious IPs detected: 1
Total failed login attempts: 5
Most aggressive IP: 192.168.1.10
Highest failed attempts from one IP: 5
Generated at: 2026-03-28 16:24:46
```

## Running the Program

Clone the repository:
```
git clone https://github.com/milindpat/cloud-log-monitoring-system.git
cd cloud-log-monitoring-system
```
Then run:
```
python src/log_monitor.py --log data/server_log.txt --threshold 5
```
Stop monitoring by pressing:
Ctrl + C
## System Workflow

1. The script continuously monitors the server log file.
2. Failed login attempts are parsed using regular expressions.
3. The system counts attempts per IP address.
4. When the threshold is reached, an alert is generated.
5. The alert is stored in log files and CSV reports.
